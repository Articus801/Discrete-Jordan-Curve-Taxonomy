# Lazy φ_n PVA

**Euler’s totient function as a compression engine.**
Multiply prime-power tuples without memory blowup. O(1) memory at scale.

## The Knockout Demo

5000 multiply operations, same workload:

|  | Lazy φ_n | Naive Append | Improvement |
| --- | --- | --- | --- |
| **Final tuples** | 3 | 5001 | **1667x smaller** |
| **Total time** | 0.0036s | 0.0140s | **3.9x faster** |
| **Time per op** | 0.72µs | 2.80µs | **4x faster** |
| **Memory scaling** | O(1) bounded | O(n) to 5000 | **∞ at scale** |

![Lazy φ_n vs Naive: Memory + Speed](docs/phi_knockout_dual.png)

Green line = Lazy φ_n with EAGER_THRESHOLD=50. Flat memory, flat speed.
Red line = Naive append. Linear growth until MemoryError.
